Basic Procedures

Laser Line Triangulation Principle

  • Projecting a laser line on the target surface
  • Diffuse reflected light of the laser line is registered by a high quality sensor array
  • Calculation of the distance coordinate Z and the actual position X along the laser line for all measuring points

Demonstration Programs and Basic Settings

The scanCONTROL devices are equipped with ethernet 100/1000MBit full duplex. All scanCONTROL demonstration programs support the communication via Ethernet interface.
Ethernet connections require the appropriate setup of the network card and firewall settings on the host PC (see installation help of the scanCONTROL software tools for further details).

All devices have a Multifunction Digital I/O Port that provides power supply, RS422 (for serial communication or master-slave operation), digital inputs (for User-Mode switching, encoder input, trigger) and external laser switch off (optional for safety regulations).

Basic parameters for measurement are profiles per second and exposure time. The default values can be used as base for parameter adaptions to the target's requirements. The best value for the exposure time depends on the reflective properties of your target. The automatic exposure control (default) tries to find automatically an appropriate exposure time for a given target. The exposure time cannot be longer than the period resulting from the profiles per second.

Factory Default Settings

After power up the scanCONTROL sensor starts profile measurements with factory settings:

  • Profile frequency: 200 profiles per second
  • Exposure Time: 100μs, automatic exposure control enabled
  • Laser Power: standard
  • Threshold for point detection: 128 (dynamic)
  • Region Of Interest Preset Number: 0 (whole sensor matrix)
  • Resolution: 2048 points per profile
  • Data transfer: container mode
  • Container mode settings:
    • Data from stripe 1
    • Information field: Z
    • Container size: 200 profiles per container (@ 200 profiles/s = 1 frame/s)

Store Settings and Recall Them at Power up

The scanCONTROL supports storing custom settings. These stored settings are called User Modes. Fifteen User Modes are available. At start up the most recently stored User Mode is recalled automatically. To store settings into User Modes or recall settings from User Modes, use one of the scanCONTROL demonstration programs.
Note: The duration to load a user mode is approx. 10-100ms (depending on the parameters that have to be changed between the modes).

Advanced Procedures

Profile Resolution

The scanCONTROL supports the following lateral resolutions in terms of point per profile: 2048, 1024, 512 and 256. A bigger setting for points per profile results in a higher processing load, a higher post-processing load (only scanCONTROL SMART sensors) and a higher Ethernet bandwidth needed for profile/container data transmission.

Operation Modes

High Resolution mode

The High Resolution mode is the standard mode for profile acquisition. All rows on the sensor matrix are used to compute the profile points and so the High Resolution mode provides profile data with the best Z linearity compared to the other modes.

High Speed mode

The High Speed mode only uses every second row on the sensor array to compute profile points. Therefore, profiles can be acquired at double profile frequency (in comparison to the High Resolution mode at the same Region Of Interest size) while losing a small degree of Z linearity.

High Dynamic Range mode (HDR)

The High Dynamic Range mode is used for target surfaces that have inhomogenous surface properties (e.g. very dark and very bright parts). The sensor rows are exposed alternately. Every uneven sensor matrix row is exposed with the set exposure time (or the exposure time deriving from the auto exposure algorithm). Every even row is exposed with the exposure time from the uneven rows times four. The sensor array image with rows of both exposure times is read out as a whole so that no in-motion unsharpness occurs between the two exposure time images.

All scanCONTROL measuring ranges have a defined calibration offset (CO) that can be read from the sensor. This calibration offset (e.g. 85mm for the 30xx-25) is close to the middle of the measuring range and is the mirror axis for digital Z flipping. See Conversion To Millimeter and the SDK documentation for details.
All ROI settings have to be done with respect to the sensor array view in the correct rotation. All scanCONTROL demonstration programs show the sensor array image in the correct rotation. If the sensor array image is grabbed directly via LLT.dll or GigE Vision, it has to be rotated by 180 degrees to get the correct orientation. See the scanCONTROL SDKs for details.

Maximum Frequencies of Profile Measurements

The maximum profile frequency depends on four basic factors:
  • Measurement target (required Exposure Time)
  • Sensor hardware (Region Of Interest, Profile Processing)
  • Post processing (only scanCONTROL SMART sensors)
  • Needed Ethernet bandwidth for data transmission
The smallest value limits the overall performance.

Measurement Target

Most targets require an exposure time between 0.1 and 1.0ms. Thus, 1000 profiles per second or more can be measured. Very dark objects like black rubber or shiny metal surfaces may require a longer exposure time or a higher laser power (class 3R, optional available for scanCONTROL sensors with red laser).

Sensor Hardware

scanCONTROL 3000 and 3010 devices can acquire up to 300 profiles per second. The speed is limited by the internal processing hardware. scanCONTROL HIGH SPEED devices can acquire up to 10000 profiles per second. The actual measuring speed depends on two basic factors and the minimum of both is the limit for this category:
  • Number of rows evaluated on the sensor array

    Formula: Max.Frequency = 1 MHz/ceil(33.25+(rows+3)*2.6875)

    Exemplary profile frequencies:

    Number of lines 1088 816 544 272 136 68 34
    Frequency 337Hz 447Hz 664Hz 1293Hz 2457Hz 4444Hz 7518Hz

    Example ROI in High Speed mode

    If the sensor is used in high-speed mode, only every second row is used for evaluation and thus the number of covered rows by the ROIs is divided by two.

    scanCONTROL 30xx sensors provide two Regions Of Interest and one Region Of No Interest (RONI) to select the best matrix region for the measurement target. The number of rows evaluated on the sensor array is the set union of the covered rows of ROI1 and ROI2 (see OpManPartB.html and the SDK documentation for details). The RONI does not have any effect on the evaluated number of sensor matrix rows.

  • Number of points per profile for processing. Following data rate limits are valid:
    • 2048 points per profile: Max.Frequency = 2703Hz
    • 1024 points per profile: Max.Frequency = 4000Hz
    • 512 points per profile: Max.Frequency = 7143Hz
    • 256 points per profile: Max.Frequency = 10000Hz

  • laserpulse mode: data rate limit Max.Frequency = 2100Hz

Post Processing

The processing time for the post processing algorithms varies according to the complexity of the post processing algorithms set up with scanCONTROL Configuration Tools. Further, the algorithms scale with the value set points per profile.

Needed Ethernet Bandwidth for data transmission

Transmission speed mainly depends on the amount of transmitted data.
In addition to the transmission of the complete profile information ("full set") scanCONTROL 3xxx sensors provide the transmission of partial profiles to reduce the required transmission bandwidth significantly. Transmitting only parts of the data set, for example profile data X/Z only without additional information, will decrease the required bandwidth even more. The frequencies above may also be reached with the container transmission only.
The Ethernet interface does not provide a bandwidth control. Packets are sent as fast as possible. To use the uncompressed Video Mode or to run the sensor at its limits, Jumbo Frames have to be active (Ethernet interface card setting supported by most Gigabit Ethernet interface cards) and all used network components (e.g. Switches) between scanCONTROL sensor and the PC. The packet size parameter has to be set to 4096 Byte.
If the used Ethernet interface card or one of the used Ethernet components between scanCONTROL sensor and the PC does not support Jumbo Frames, the packet size has to be set to 1024. In this case not all listed profile frequencies can be reached.

Remarks
  • Hubs must not be used.
  • If a switch is used, a 1000MBit switch is strongly recommended.
  • If a switch is used, the packet delay parameter should be set. Switches should not be cascaded (the packet delay parameter is only valid for the switch that the scanCONTROL sensor is connected to).
  • We recommend using Jumbo Frames (Ethernet interface card setting) to use the Video Mode in full resolution and to reach the maximum profile frequencies.
  • Only about 80% of the available Ethernet bandwidth (100MBit or 1000MBit) should be dedicated for the profile transmission of all sensors.
  • A single sensor can send up to 800MBit/s.

Profile Processing

Threshold:
The recommended value 128 is applicable for most measurement targets. It can be increased for target surfaces that deliver fuzzy reflections on the sensor array to get stable measurement points. If the target has different surface properties (e.g. bright and dark surface regions), the Threshold Automatic feature should be used to get good profile data of all surface regions. It is not recommended to decrease the absolute threshold to get more reflections on dark surfaces (ambient light might influence the measurements).

Multiple reflections:
The digital hardware can recognize up to 4 reflections per sensor matrix column. Only valid reflections will be processed in the result set (according to Threshold and Peak Filter settings). According to the reflection selection parameter the scanCONTROL sensor selects the main reflection to generate the corresponding profile point. The selection is based on the position (first or last), the integral intensities (largest area) or the maximum intensities of the valid reflections.

Digital I/O Port

scanCONTROL sensors have a programmable Multifunction Digital I/O-Port. The port provides one RS422 half-duplex channel for serial communication or synchronization and four general purpose inputs with programmable voltage level (TTL/5V or HTL/24V) and selectable Pull up or Pull down behavior. For details please refer to the Digital I/O Configuration.

Sensor LEDs and Error Codes

  1. The Laser LED indicates the state of the laser (on/off)
  2. The Data LED flashes green.
