Demonstration Programmes and Basic Settings

The scanCONTROL devices are equipped with ethernet 100/1000MBit full duplex. 

All devices have also a Multi Function Port that can be used as a serial communication port. All demonstration programmes support the communication via the high speed interface. An ethernet connection requires the appropriate setup of the network card and firewall settings on the host PC (see installation help for further details).

Basic parameters for measurement are profiles per second and shutter. Choosing too many profiles per second can freeze your computer. The default values are good for evaluation.

The optimum value of the shutter time strongly depends on the reflective properties of your target. The automatic shutter control (default) tries to find automatically an appropriate value for a given target.

Shutter times longer than 10ms are not recommended. The shutter time cannot be longer than the period resulting from the profiles per second.

Factory Default Settings

After power up the scanCONTROL starts profile measurements with factory settings:

  • 100 profiles per second
  • 1280 points per profile
  • 100μs shutter time, automatic shutter control
  • Standard laser power
  • Measuring field "large" (index 0), this covers the standard measurement range given in the data sheet
If there are user modes stored then the most recently stored user mode will be loaded instead of the factory settings.

Store Settings and Recall Them at Power up

The scanCONTROL supports storing custom settings. These stored settings are called user modes. Fifteen user modes are available. At start up the most recently stored user mode is recalled automatically. To store settings into user modes or recall settings from user modes, use one of the demonstration programmes.
Remark: The duration to load a user mode is approx. 10ms.

Profile Resolution

The scanCONTROL supports the following lateral resolutions in terms of point per profile: 1280, 640, 320 and 160. Changing the lateral resolution allows to select the trade off point between lateral resolution and distance resolution.

Maximum Frequencies of Profile Measurements

The maximum profile frequency depends on three basic factors:
  • Measurement target
  • Sensor hardware
  • Transmission bandwidth
The smallest value limits the over all performance.

Measurement Target

Most targets require a shutter time between 0.1 and 1.0ms. Thus, 1000 profiles per second or more can be measured. Very dark objects like black rubber or shiny metal surfaces may require a longer shutter time or a higher laser power (class 3B, optional available).

Sensor Hardware

scanCONTROL 2900 and 2910 devices can acquire up to 300 profiles per second. The speed is limited by the internal processing hardware.
scanCONTROL 2950 and 2960 devices contain faster processing hardware. They can acquire up to 2000 profiles per second. A profile can have up to 1280 points. The measurement field setting allows a fine grained adjustment between speed and field of view.

Transmission Bandwidth

Transmission speed mainly depends on the amount of transmitted data. The below values apply if the complete data set is transmitted. Transmitting only parts of the data set, for example profile data only without additional information, will decrease the required bandwidth. The frequencies above may be reached with the container transmission only.

The ethernet interface does not provide a bandwith control. Packets are sent as fast as possible. Furthermore, most ethernet cards support only packets up to approximately 1500bytes. Therefore, it is recommended to always use 1024bytes/packet for the ethernet interface.
Required bandwith per sensor running at 100 profiles per second
160 points per profile 320 points per profile 640 points per profile 1280 points per profile
9MBit/s 18MBit/s 36MBit/s 72MBit/s

Remarks:

  • It is recommended to connect each device directly to one dedicated network card.
  • Hubs must not be used.
  • If a switch is used, the packet delay parameter should be set. Switches must not be cascaded.
  • Only about 80% of the available bandwidth (100MBit or 1000MBit) should be dedicated for the profile transmission of all sensors.
  • A single sensor can send up to 400MBit/s

Processing of Profile Data

Threshold:
The recommended value 128 can be decreased/increased for special situations (for instance decreased for black rubber targets). Reflections must have a higher intensity than the threshold level which was adjusted. If suppress ambient light is checked threshold is relativ to the ambient light (recommended).

Video filter:
Slight smoothing the video signal before processing reflections (recommended).

Use calibration:
Convert image sensor row/column numbers to millimeters. Recommended in most cases.

Multiple reflections:
The digital hardware can recognize up to 4 reflections per image row. They all can be processed in case of later decision on which reflection belongs to the real target. This decision can also be made by the scanCONTROL based on the positions, the integral intensities or the maximum intensities of reflections. This procedure saves time because the decision is made first, high resolution and calibration algorithms are performed later for less reflections.

Multi-Function Port

The scanCONTROL features a programmable Multi-Function Port. The port provides 1 half-duplex channel for serial communication or synchronisation. And the port provides three general purpose inputs with programmable voltage level Low Level Logic (TTL, 5V) or High Level Logic (HTL, 24V). For details please refer to Multi-Function Port Configuration.
