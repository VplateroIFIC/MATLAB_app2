MEDAQLib.lvlib:SensorCommand.vi

ERR_CODE SensorCommand (DWORD instanceHandle);

Before sending a command to the sensor the command and parameters must

be specified with SetParameter... functions. After the command is executed the

answer can be read using GetParameter... functions. At start of this function,

all internal parameters starting with SA_, and IA_ are cleard (like at function

ClearAllParameters). At end of this function, all internal parameters starting

with SP_, CP_, and IP_ are cleared (like at function ClearAllParameters).

 instanceHandle

Parameter: DWORD instanceHandle

Direction: [IN]

Description: Number of the sensor instance, previously returned by

CreateSensorInstance.

Fehler (Eingang, kein Fehler)

Fehler (Eingang) kann Fehlerangaben der vorherigen VIs aufnehmen. Auf Grundlage dieser Angaben kann entschieden werden, ob bei Entstehen eines Fehlers in einem anderen VI eine bestimmte Funktion übersprungen werden soll.

Für eine ausführlichere Erläuterung des Fehlers klicken Sie mit der rechten Maustaste auf das Anzeigeelement Fehler (Eingang) und wählen Sie aus dem Kontextmenü Fehler beschreiben oder Warnung beschreiben aus.

    		status

    Status ist TRUE (rotes Kreuz), wenn ein Fehler auftritt, andernfalls FALSE (grüner Haken).

    Für eine ausführlichere Erläuterung des Fehlers klicken Sie mit der rechten Maustaste auf das Element Fehler (Eingang) und wählen Sie aus dem Kontextmenü Fehler beschreiben oder Warnung beschreiben aus.

    		code

    Code ist der Fehler- oder Warnungscode.

    Für eine ausführlichere Erläuterung des Fehlers klicken Sie mit der rechten Maustaste auf das Element Fehler (Eingang) und wählen Sie aus dem Kontextmenü die Option Fehler beschreiben oder Warnung beschreiben aus.

    		source

    Quelle beschreibt den Ursprung des Fehlers oder der Warnung.

    Für eine ausführlichere Erläuterung des Fehlers klicken Sie mit der rechten Maustaste auf das Element Fehler (Eingang) und wählen Sie aus dem Kontextmenü Fehler beschreiben oder Warnung beschreiben aus.

DLL-Path

INPUT: Path to library "MEDAQLib.dll".

ERR_CODE

Returns:

ERR_NOERROR (0) on success.

ERR_INSTANCE_NOT_EXTST (-24) if instanceHandle is not valid.

ERR_NOT_OPEN (-3) if connection is not established

ERR_UNKNOWN_SENSOR_COMMAND (-14) if parameter "S_Command" is not valid.

ERR_BAUDRATE_TOO_LOW (-22) if at sensor IFD2430 over RS232 with Baudrate<38400 the CCD line (Get_CCD) or other binary data (Get_DarkSig, Get_...) should be read.

ERR_APPLYING_PARAMS (-4) if interface parameters changed by command (e.g. change baudrate) and cannot be applied.

ERR_NO_ANSWER_RECEIVED (-20) if no answer but other data is received within answer time.

ERR_TIMEOUT_READING_FROM_SENSOR (-8) if no answer is received within answer time.

ERR_CLEARING_BUFFER (-6) for command "Clear_Buffers" if specified device could not be cleared.

ERR_NOMEMORY (-19) if a data buffer could not be allocated.

ERR_UNKNOWN_SENSOR_ANSWER (-15) if sensor answer is not an answer to a command.

ERR_SENSOR_ANSWER_ERROR (-16) if sensor answer cannot be interpreted or sensor returned an error number.

ERR_SENSOR_ANSWER_TOO_SHORT (-17) if sensor answer is too short.

ERR_SENSOR_ANSWER_DOES_NOT_MATCH_COMMAND (-21) if answer does not match the command.

ERR_SENSOR_ANSWER_WARNING (-27) if sensor returned a warning.
