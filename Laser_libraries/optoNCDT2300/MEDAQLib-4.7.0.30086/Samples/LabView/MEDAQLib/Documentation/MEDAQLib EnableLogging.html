MEDAQLib.lvlib:EnableLogging.vi

ERR_CODE EnableLogging (DWORD instanceHandle, BOOL enableLogging, int

logType, int logLevel, LPCSTR logFile, BOOL logAppend, BOOL logFlush);

Set the parameters to enable logging.

 instanceHandle

Parameter: DWORD instanceHandle

Direction: [IN]

Description: Number of the sensor instance, previously returned by CreateSensorInstance.

logLevel

Parameter: int logLevel

Direction: [IN]

Valid values:

A bit combination of following values:

1= EMERGENCY_LEVEL (logging emerging errors)

2= CRITICAL_LEVEL (logging critical errors)

4= ERROR_LEVEL (logging errors which occurs)

8= WARNING_LEVEL (logging warnings from MEDAQLib)

16= NOTICE_LEVEL (logging notices)

32= TRACE_LEVEL (logging function calls)

64= DATA_LEVEL (logging data in binary mode)

Description: This paramenter specifies the kind of event to log.

Fehler (Eingang, kein Fehler)

Fehler (Eingang) kann Fehlerangaben der vorherigen VIs aufnehmen. Auf Grundlage dieser Angaben kann entschieden werden, ob bei Entstehen eines Fehlers in einem anderen VI eine bestimmte Funktion übersprungen werden soll.

Für eine ausführlichere Erläuterung des Fehlers klicken Sie mit der rechten Maustaste auf das Anzeigeelement Fehler (Eingang) und wählen Sie aus dem Kontextmenü Fehler beschreiben oder Warnung beschreiben aus.

    		status

    Status ist TRUE (rotes Kreuz), wenn ein Fehler auftritt, andernfalls FALSE (grüner Haken).

    Für eine ausführlichere Erläuterung des Fehlers klicken Sie mit der rechten Maustaste auf das Element Fehler (Eingang) und wählen Sie aus dem Kontextmenü Fehler beschreiben oder Warnung beschreiben aus.

    		code

    Code ist der Fehler- oder Warnungscode.

    Für eine ausführlichere Erläuterung des Fehlers klicken Sie mit der rechten Maustaste auf das Element Fehler (Eingang) und wählen Sie aus dem Kontextmenü die Option Fehler beschreiben oder Warnung beschreiben aus.

    		source

    Quelle beschreibt den Ursprung des Fehlers oder der Warnung.

    Für eine ausführlichere Erläuterung des Fehlers klicken Sie mit der rechten Maustaste auf das Element Fehler (Eingang) und wählen Sie aus dem Kontextmenü Fehler beschreiben oder Warnung beschreiben aus.

DLL-Path

INPUT: Path to library "MEDAQLib.dll".

enableLogging

Parameter: BOOL enableLogging

Direction: [IN]

Valid values:

0= FALSE

1= TRUE

Description: This paramenter enables or disables logging to file for

debugging purposes.

logAppend

Parameter: BOOL logAppend

Direction: [IN]

Valid values:

0= FALSE

1= TRUE

Description: This paramenter specifies if the logfile should be cleared at

opening or if the new data should be appended to file.

logFlush

Parameter: BOOL logFlush

Direction: [IN]

Valid values:

0= FALSE

1= TRUE

Description: This paramenter specifies if the logfile should be flushed

after each output. In this case, it is sure that all information is stored

to logfile before proceeding. But depending on the storage device it

can slow down the MEDAQLib.

logType

Parameter: int logType

Direction: [IN]

Valid values:

A bit combination of following values:

1= HIGH_TYPE (User <--> MEDAQLib)

2= MIDDLE_TYPE (Sensor layer <--> Interface layer)

4= LOW_TYPE (MEDAQLib <--> Hardware driver)

8= ERROR_TYPE (Any errors reported by MEDAQLib)

16= DRIVER_TYPE (Hardware driver <--> Sys driver)

Description: This paramenter specifies the type of messages to log.

logFile

Parameter: LPCSTR logFile

Direction: [IN]

Description: File name of log file.

ERR_CODE

Returns:

ERR_NOERROR (0) on success.

ERR_INSTANCE_NOT_EXTST (-24) if instanceHandle is not valid.

ERR_NOT_OPEN (-3) if connection is not established

ERR_NO_SENSORDATA_AVAILABLE (-13) if the receive buffer does not contain any value from sensor.

instanceHandle

Parameter: DWORD instanceHandle

Direction: [OUT]

Description: Number of the sensor instance, copied from input.

DLL-Path

OUTPUT: Path to library "MEDAQLib.dll".
